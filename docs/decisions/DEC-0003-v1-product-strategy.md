# DEC-0003 - Approve V1 Product Strategy

Status: `proposed`

Date proposed: 2026-07-02

Approving authority: user

## Context

The approved Vision defines a personal investment committee centered on cited
evidence, explicit uncertainty, thesis memory, and human decision authority.
Gate 2 must convert that vision into a bounded first usable release before any
milestone or technology decision is authorized.

## Proposed Decision

Approve `docs/PRODUCT_STRATEGY.md` as the V1 product contract with these locked
decisions:

- public equities across Indonesia and the United States
- up to 100 tracked companies tagged `Owned` or `Watchlist`
- first-class existing-thesis and company/sector exploration entry paths
- guided hybrid evidence with official-source priority and labeled fallback
  sources
- temporary, unranked sector shortlists requiring explicit save
- on-demand review before scheduled monitoring
- weekly top-10 priority queue plus full tracked-company index
- thesis-state outcomes and optional user-originated investment actions
- one usable release delivered through multiple approved vertical milestones
- four weekly dogfood cycles followed by a 3–5 user private beta

## Boundaries

This decision does not approve trade execution, system-generated investment
actions, ranked recommendations, position sizing, brokerage data, private
assets, collaboration, scheduled monitoring, architecture, schemas,
dependencies, APIs, code, or deployment.

## Options

1. Accept the strategy and authorize creation of the first milestone packet.
2. Request targeted strategy changes while keeping Gate 2 proposed.
3. Reject the V1 release definition and return to product-strategy drafting.

## Consequences If Accepted

- `docs/PRODUCT_STRATEGY.md` becomes the approved Gate 2 authority.
- `ACTIVE_MILESTONE.md` may point to a proposed M001 packet.
- Only milestone planning becomes authorized; evaluation, architecture, and
  implementation remain gated.
