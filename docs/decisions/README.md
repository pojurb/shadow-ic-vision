# Decision Records

Record approved product and architecture decisions here using stable identifiers:

- `DEC-####-*.md` for product, policy, and risk-acceptance decisions
- `ADR-####-*.md` for architecture decisions

Each record includes:

1. Status: proposed, accepted, superseded, or rejected
2. Date and approving authority
3. Context and decision required
4. Options considered
5. Decision and rationale
6. Consequences and risks
7. Reversal, migration, or supersession path
8. Affected playbook, milestone, evaluation, and implementation files

Vision approval must be explicit and recorded here before milestone planning is
authorized.
